# **SentriZK Authentication - Frontend Developer Guide**

## **Overview**

This authentication system uses **Zero-Knowledge Proofs (ZKP)** to allow users to register and log in **without revealing their secret credentials**. Each user has:

* **Commitment:** A hash derived from their secret (like a wallet private key).
* **Nonce:** A one-time random value issued by the server for login sessions.
* **Session:** A derived value for session validation using Poseidon hash.

Key points for frontend developers:

1. Users **never send their secret** directly.
2. All authentication involves **generating ZKP proofs** using the secret and server-provided nonce.
3. Login is **resistant to replay attacks**.

## **API Endpoints**

### **1. Health Check**

* **URL:** `/health`
* **Method:** `GET`
* **Response:**

```json
{ "ok": true }
```

Purpose: Verify the server is running.

---

### **2. Check Username Availability**

* **URL:** `/check-username/:username`
* **Method:** `GET`
* **Response:**

```json
{ "available": true } // or false
```

**Usage:** Call before registration to ensure username is free.

---

### **3. Fetch Commitment + Nonce (Anti-Replay)**

* **URL:** `/commitment/:username`
* **Method:** `GET`
* **Response:**

```json
{
  "username": "alice",
  "commitment": "1234567890",
  "nonce": "9876543210"
}
```

**Frontend Flow:**

1. Call this endpoint before login.
2. Receive the **nonce** to generate a **login proof**.
3. The nonce is valid **only once**, preventing replay attacks.

---

### **4. Registration**

* **URL:** `/register`
* **Method:** `POST`
* **Request Body:**

```json
{
  "username": "alice",
  "proof": {...},       // ZKP proof generated by the frontend
  "publicSignals": [...] // Signals from the ZKP circuit (first element is commitment)
}
```

* **Response (Success):**

```json
{
  "status": "ok",
  "username": "alice",
  "commitment": "1234567890"
}
```

**Frontend Flow:**

1. Generate the **commitment** from the user’s secret.
2. Generate a **ZKP proof** using the commitment.
3. Submit proof and publicSignals to `/register`.
4. Check response status.

---

### **5. Login**

* **URL:** `/login`
* **Method:** `POST`
* **Request Body:**

```json
{
  "username": "alice",
  "proof": {...},       // ZKP proof using commitment + nonce
  "publicSignals": [
      "commitment",
      "session"         // Poseidon(commitment, nonce)
  ]
}
```

* **Response (Success):**

```json
{
  "status": "ok",
  "message": "Login successful",
  "session": "9876543210"
}
```

**Frontend Flow:**

1. Request **commitment + nonce** from `/commitment/:username`.
2. Generate a **login proof** using:

   ```
   ZKP(secret, nonce)
   ```
3. Send proof + publicSignals to `/login`.
4. If successful, store **session** for further authenticated requests.

---

## **Security Notes for Frontend**

1. **Never send the secret directly**—always use ZKP to prove knowledge.
2. **Nonce changes each login**; do not reuse old nonces.
3. **Commitment alone is useless** without the secret and proper proof.
4. Treat **session** as a temporary login token; verify with server if needed.

---

## **Example Frontend Flow (Pseudo-JS)**

```js
// 1. Check username availability
const res = await fetch(`/check-username/${username}`);
const { available } = await res.json();
if (!available) throw new Error("Username taken");

// 2. Register
const regProof = generateZKP(secret); // your frontend ZKP lib
const regSignals = getPublicSignals(secret);
await fetch('/register', {
  method: 'POST',
  body: JSON.stringify({ username, proof: regProof, publicSignals: regSignals }),
  headers: { 'Content-Type': 'application/json' }
});

// 3. Login
const commitRes = await fetch(`/commitment/${username}`);
const { commitment, nonce } = await commitRes.json();

const loginProof = generateLoginZKP(secret, nonce);
const loginSignals = [commitment, poseidon(commitment, nonce)];

const loginRes = await fetch('/login', {
  method: 'POST',
  body: JSON.stringify({ username, proof: loginProof, publicSignals: loginSignals }),
  headers: { 'Content-Type': 'application/json' }
});
const { session } = await loginRes.json();
```

## **Frontend Developer Checklist**

* [ ] Integrate a **ZKP library** (like `snarkjs`) in the frontend.
* [ ] Always fetch **nonce before login**.
* [ ] Generate proofs **offline in browser**; never expose secrets.
* [ ] Handle server errors (`commitment mismatch`, `session mismatch`, `invalid proof`) gracefully.
* [ ] Store session token in memory (or secure storage) for authenticated requests.

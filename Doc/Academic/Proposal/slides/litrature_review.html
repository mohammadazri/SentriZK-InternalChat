Literature Review Matrix

A comprehensive 6-column analysis covering Secure Messaging, Zero-Knowledge Proofs, and AI Anomaly Detection.

01 Secure Messaging & Insider Threats

Reference Focus & Problem Statement Method Findings Strength / Weakness Future Direction
Alatawi, M., & Saxena, N. (2023). SoK: An analysis of end-to-end encryption and authentication ceremonies in secure messaging systems. ACM Asia Conference on Computer and Communications Security.
Focus: Usability of authentication ceremonies (QR/Keys) in 17 apps.

Problem: "Opportunistic" encryption fails against MitM. Manual ceremonies are optional and hard to find. 		SoK & Usability Tests All apps vulnerable to active MitM by default. Users consistently fail to verify long strings manually. Strength Comprehensive scope (17 apps). Clearly identifies human error. Weakness Suggests recommendations rather than building a technical solution. Need for automated or semi-automated authentication ceremonies to remove user burden.
Srinivasan, K., et al. (2025). Insider threat detection and secure data transfer leveraging bidirectional LSTM. IEEE Access.
Focus: Detecting malicious insiders using Bi-LSTM.

Problem: Insiders bypass firewalls; standard methods miss subtle behavioral changes. 		Bi-LSTM (Deep Learning) Bi-LSTM understands "context" of actions over time, achieving higher accuracy than standard ML. Strength High accuracy for time-series log data. Weakness Computationally heavy; usually requires a server (not ideal for edge). Improving training speed and testing on diverse datasets.
Alzaabi, F. R., & Mehmood, A. (2024). A review of recent advances, challenges, and opportunities in malicious insider threat detection using machine learning methods. IEEE Access.
Focus: Survey of ML techniques for insider threats.

Problem: No single perfect algorithm exists; balancing False Positives vs False Negatives is difficult. 		Systematic Literature Review (SLR) Deep Learning is the standard but lacks explainability. Data imbalance hinders training. Strength Excellent state-of-the-art overview. Weakness Does not propose a new tool, only reviews existing ones. Need for "Explainable AI" (XAI) and privacy-preserving detection methods.
Ilesanmi, M. (2025). Threat modeling for enterprise secure messaging applications. Journal of Cybersecurity and Privacy.
Focus: Security holes in apps like Slack/Teams.

Problem: Complexity (File/Video/Chat) creates attack surfaces encryption cannot fix. 		Threat Modeling (STRIDE) E2EE protects transit, but metadata and endpoint security remain vulnerable. Strength Very specific to the chat domain. Weakness Focuses on finding problems rather than building solutions. Recommends better endpoint verification and minimizing metadata collection.

02 Zero-Knowledge Proofs (Authentication)

Reference Focus & Problem Statement Method Findings Strength / Weakness Future Direction
Bhattacharya, S., et al. (2024). Enhancing digital privacy: The application of zero-knowledge proofs in authentication systems. International Journal of Information Security.
Focus: ZKPs (SNARKs/STARKs) to replace passwords.

Problem: Centralized databases create single points of failure for credential theft. 		Comparative Analysis ZKPs verify users without data transmission. STARKs avoid "trusted setup" but have larger proofs. Strength Clearly distinguishes zk-SNARK vs zk-STARK. Weakness High complexity and specialized knowledge barriers. Developing user-friendly frameworks/APIs to lower entry barrier.
Chen, Z., et al. (2023). A survey on zero-knowledge authentication for Internet of Things. Electronics, 12(5).
Focus: ZKP on weak devices (IoT/Mobile).

Problem: Standard ZKP is too heavy/slow for devices with low battery/memory. 		Survey & Comparative Analysis Non-interactive ZKP and zk-SNARKs (Groth16) are best for low-power/unstable internet. Strength Validates lightweight ZKP on restricted hardware. Weakness Focuses on IoT sensors, not specifically smartphones. Optimizing the "Setup Phase" to be less dependent on trusted third parties.
Luong, D. A., & Park, J. H. (2023). Privacy-preserving identity management system on blockchain using zk-SNARK. IEEE Access.
Focus: Self-Sovereign Identity via Blockchain + ZKP.

Problem: "Log in with Google" tracks user activity; users lose privacy. 		System Implementation Successfully verified users without storing passwords. Gas costs (fees) were the main bottleneck. Strength Practical implementation details (code/tools). Weakness Blockchain focus might be "overkill" for simple chat. Reducing computational cost of generating proofs on the client side.
[Authored Group]. (2025). A secure framework for privacy-preserving analytics in healthcare records using zero-knowledge proofs. Journal of Medical Systems.
Focus: ZKP for sensitive medical records.

Problem: Privacy laws (HIPAA/GDPR) make sharing data for analysis difficult. 		Framework Design ZKP allows "Auditability without Visibility"—checking math without seeing data. Strength Demonstrates high-stakes trust in ZKP. Weakness Complex setup required for multiple organizations. Integrating this with Federated Learning.

03 AI Anomaly Detection (On-Device)

Reference Focus & Problem Statement Method Findings Strength / Weakness Future Direction
Natha, S. (2022). A systematic review of anomaly detection using machine and deep learning techniques. Quaid-e-Awam University Research Journal.
Focus: ML vs DL review for anomalies.

Problem: "Unknown nature of anomalies" and noise make static algorithms (SVM) fail. 		Systematic Literature Review Deep Learning outperforms ML on high-dimensional data but requires significant bandwidth/time. Strength Specific critique of One-Class SVM (underfitting). Weakness High accuracy comes at high computational cost. Improve performance in limited resource environments (IoT/Mobile).
Gaurav, A., et al. (2025). AI-driven smishing detection in Android devices using TinyBERT and Aquila optimization. Computers & Security.
Focus: Smishing detection on Android.

Problem: Rule-based filters miss context. Standard BERT is too heavy for phones. 		Hybrid DL (TinyBERT + Aquila) 96.81% accuracy with high precision on Android; effective at reducing false positives. Strength High accuracy specifically tailored for Android. Weakness Transformer models can still be battery-heavy. Integrating detection engine into real-time prevention systems.
Bispo, G. D., et al. (2025). PHILDER: Lightweight framework for intelligent phishing detection on resource-limited devices. IEEE Internet of Things Journal.
Focus: Edge Computing phishing detection.

Problem: Cloud detection causes Latency and Privacy Risks (URL leakage). 		Experimental (ARM Architecture) High accuracy (90%+) with <100ms latency. Proved heavy cloud servers aren't needed. Strength Exact validation for local/edge AI modules. Weakness Limited to URL detection (no files). Expanding model to detect SMS phishing and QR codes.
Karim, A., et al. (2023). Phishing detection system through hybrid machine learning based on URL. IEEE Access.
Focus: Hybrid ML for URL analysis.

Problem: Single algorithms (Random Forest/SVM) have blind spots. 		Hybrid Model Experiment Hybrid models extract lexical features (e.g., special chars) better than single models. Strength Provides specific features to look for (lexical). Weakness Hybrid models can be slower/larger. Real-time updating of the model as new phishing trends emerge.

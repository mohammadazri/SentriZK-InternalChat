Domain 02

Secure Messaging & Insider Threats

Literature Review
Approach Literature Findings Comparative Verdict
Method A:
Manual Verification		 Standard practice (QR codes/Safety Numbers). Analysis of 17 apps shows users consistently ignore these "ceremonies," leaving encryption vulnerable to active MitM attacks. Alatawi & Saxena (2023) REJECTED
Relies heavily on human diligence, which is the weakest link in security.
Method B:
Server-Side Deep Learning		 Uses Bi-LSTM networks to scan user logs. While highly accurate, it requires centralized data processing, which inherently violates user privacy. Srinivasan et al. (2025); Alzaabi (2024) REJECTED
Compromises privacy by requiring a central server to analyze behavior.
Method C:
Automated Endpoint Security		 Focuses on securing the device (endpoint) itself. Threat modeling proves the endpoint is the primary attack surface, even if transmission is encrypted. Ilesanmi (2025) SELECTED (SentriZK)
Moves defense away from the user (automation) and away from the server (privacy).
